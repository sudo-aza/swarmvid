### Bugs found

**Bug 1 (CRITICAL): render_scene.py alpha compositing broken**

The script creates an `Image.new("RGBA")` and draws multiple elements with alpha fill values:
- Line 77: particles `fill=(255, 255, 255, alpha)` — intended as subtle transparent dots
- Line 113: header bar `fill=(0, 0, 0, 100)` — intended as semi-transparent overlay
- Lines 144-147: text box `fill=(0, 0, 0, 80)` — intended as semi-transparent backdrop
- Line 161: text fade-in `fill=(240, 240, 240, line_alpha)` — intended as gradual opacity
- Line 176: source text `fill=(150, 150, 150, 180)` — intended as subtle text

But line 178 calls `img.convert("RGB")` which **drops the alpha channel entirely**. Verified with a test: a pixel drawn as `(255, 255, 255, 50)` becomes `(255, 255, 255)` after conversion — fully opaque white.

Result: ALL transparency effects are broken. Particles will be solid white dots, overlays will be solid black, text fade-in won't work. The only correct alpha compositing in the script is the title card (lines 121-132) which properly uses `Image.alpha_composite()`.

Reported as BLACKBOARD Task #6.

**Bug 2 (MEDIUM): assemble_video.py crossfade parameter is a lie**

The function signature accepts `crossfade_s=1.0` and the docstring says "with crossfade transitions." But line 28's comment admits "no crossfade for reliability" and the parameter is never referenced in the function body. The actual implementation uses `ffmpeg -f concat` which produces hard cuts between scenes.

Per the video spec, "smooth scene transitions" are required. Either implement crossfades or remove the misleading parameter/docstring.

Reported as BLACKBOARD Task #7.
